# G2 조직 구조 스냅샷 (CAR OrgDev 입력 — 단일 진실 원천)

> **이 문서의 용도:** CAR(Chief of AI-agent Resources)이 G2 조직을 **교육·개발·개선**할 때 벤치마킹하는 **현재 조직의 정본(snapshot).**
> CAR은 이 스냅샷 ↔ 외부 베스트프랙티스(수확된 multi-agent/오케스트라 지식)를 대조해 **개선 제안**을 만든다.
> **출처(이 스냅샷의 진실):** G2 루트 `CLAUDE.md`(유일한 헌법) + `wiki/wiki/agents/org-chart.md`. 스냅샷은 stale될 수 있으니 CAR은 큰 변경 제안 전 원본을 1회 확인.
> **갱신:** 2026-06-30 (Alpha). G2 헌법 개정 시 Alpha가 이 파일 동기화.

## 0. 한 줄 정의

회장 1명 + Alpha(CEO) + 실행 전문가 — **Flat 2단 + 검증 하드게이트** 체제. 그 위에 자동화 5개 층이 얹혀 있다. 품질은 *계층*이 아니라 *검증 게이트*가 만든다.

## 1. 조직 계층 (Flat 2단)

```
회장 (강경구)
  ├── CAE (Chief Audit Executive) — 회장 직속 독립 감사. Alpha조차 감사 대상. ★C레벨 비동급
  │     └── g2-auditor — 프로세스 감사 집행 (주기·격리 컨텍스트·수정권한 없음)
  └── Alpha (CEO) — 유일 창구 · 의도 수호 · 라우팅 · 최종 검수 (코드 직접 안 짬, 판단=opus)
        ├── 실행 전문가 (1홉 직행 — 중간 라우터 없음)
        │     ├── g2-cto (opus) — 코딩 기본값·복합
        │     ├── g2-frontend-engineer — UI·디자인 구현
        │     ├── g2-backend-engineer — API·DB·CF Worker
        │     ├── g2-clo — 법무(약관·개인정보·규제) + Mike /legal-*
        │     ├── g2-qa-tester — 독립 동작 검증 (중형+ 의무)
        │     ├── Dev Code Reviewer — 코드·보안 리뷰 (민감 작업 의무)
        │     ├── Dev Rapid Prototyper — 72h MVP
        │     ├── researcher — 리서치
        │     └── CAR (Chief of AI-agent Resources) ★ — AI 에이전트 자원 + 조직개발(OrgDev) 담당
        │           · 도구 트랙: 유튜브/깃허브 → G2 스킬·MCP 적용·검증·보고·자가개선
        │           · 조직 트랙(신규): 이 스냅샷 ↔ 외부 조직 베스트프랙티스 대조 → 조직 개선 제안(propose-only)
        │           · 페르소나: Desktop/ARD/agents/CAR.md / 시스템: ARD/ard-loop/
        └── C레벨 8종 (CTO·CFO·COO·CMO·CDO·CISO·CLO·CSO) = 사고 렌즈·자동주입 채점표
              (소환 아님. 라이브 소환은 교차기능 결정만 — 1홉·체인 금지)
```

**핵심 원칙:**
- **Alpha = 임원.** 회장과 유일 창구. 라우팅·최종 검수. 코드 직접 안 짬.
- **실행 전문가 = 실무.** 1홉 직행. 중간 관리 계층(C레벨 라우터) 없음 — 핸드오프마다 콜드스타트 = 토큰만 곱하고 스킬은 안 더함.
- **검증 게이트 = 품질.** 증빙 없는 완료는 존재하지 않는다.
- **지식 소유 원칙:** 도메인 지식은 전문가가 소유, Alpha는 "무엇을 누구에게" 라우팅 표만 쥔다(얇게 유지 = 토큰↓·망각↓).

## 2. 라우팅 — Single-Agent First (2026-06-12 증거 기반 재판정)

**기본값 = Alpha 직접 + 검증 게이트.** 작고 자기완결적이고 변경 지점이 명확하면 Alpha 직접(핸드오프 손실 0·토큰 최소). "직접 했으니 검증 면제"는 **없음.**

**위임 트리거 — 하나라도 켜질 때만 소환 (없으면 Alpha 직접):**

| 트리거 | 조건 | 처리 |
|--------|------|------|
| T1 탐색 과중 | 원인 불명 / 5+ 파일 조사 / 큰 코드베이스 수색 | 격리 실행자 1명 |
| T2 병렬 가능 | 3+ 진짜 독립 덩어리·읽기 위주 | 병렬 탐색자 |
| T3 위험 | 인증·결제·보안·아키텍처 변경 | 실행자 + 독립 리뷰 |
| T4 대형 | 신규 화면/기능 = 신규 코드 다량 | 실행자 + QA |
| T5 지식 과중 | 깊은 도메인 지식(법률·재무·보안·규제) | 도메인 소유 전문가 소환 |

**근거(2026 프로덕션 합의):** 위임 토큰 비용 — 오케스트레이터 +285%, 유연 패턴 +515%. 순차 추론에선 멀티에이전트 품질 −39~70%. 위임은 의도의 손실 압축. 단일에이전트 정확도 45%↑ 작업에 에이전트 추가 = 음의 수익.

## 3. 검증 하드게이트 (완료의 정의 = 증빙 첨부)

| 규모 | 게이트 |
|------|--------|
| 소형 (4~20줄) | 실행자 자가 검증 증빙 |
| 중형+ (20줄+/다중 파일/신규 화면/API) | + g2-qa-tester 독립 검증 |
| 민감 (인증·결제·보안·아키텍처·코어 엔진) | + Dev Code Reviewer 리뷰 (🔴블로커=구현 복귀) |

- **증빙 유형별:** 코드=빌드/테스트 실행 출력 / UI=before·after 스크린샷+console 0 / API=실호출(성공1+에러1) / 배포=라이브 URL 실접속.
- **증빙은 완료형만**(선언형·"~예정" 금지). **위험 가중 독립성:** 🔒 유형(사진·개인정보·인증·결제·보안·코어 엔진)은 소형이어도 독립검증 강제 — Alpha 재량 아님.

**§1.5 도메인 주입 매트릭스 (작업 분류 직후 Alpha 필수 룩업):** 기능 유형 → 자동 주입되는 C레벨 채점표. 예: 인증=CISO+CLO, 결제=CISO+CLO+CFO, 코어엔진=CTO+CSO+정확성 오라클, 개인정보/사진=CISO+CLO, 모든 코딩=CTO+CSO.

## 5. 위에 얹힌 자동화 5개 층 (토큰 거의 0, 훅/스킬 기반)

1. **스킬 자기개선 루프** — find-skills 자동 발동(처음 기술·2회+ 막힘) / 스킬 자동생성(`g2_origin: agent-created`) / 워크플로 자동관찰(5회+ 반복→`auto-*`) / 큐레이터(7일·미사용 리포트, 자동삭제 없음) / Record&Replay(`rec-*`).
2. **Spec-Driven Dev (Spec Kit)** — 신규 화면/기능·다중파일(T4급)엔 명세→계획→작업분해→구현 강제. 소형·버그픽스 제외(효율 원칙 보호).
3. **도구 자동매핑** — 작업 분류 직후 "작업 유형→도구" 표 룩업, 회장 언급 없이 자동 포함. 검증=`tool_usage_audit.py`(0회 호출=라우팅 실패 탐지).
4. **[확인]/[추정] 라벨** — 위험·법률·수치·단정엔 라벨 필수. [확인]=이 세션 도구 결과에 묶일 때만.
5. **보고 채널 — 폰 푸시** — 회장 모바일-우선. Telegram 푸시(`car_notify.py`). 대안=Slack MCP.

## 7. 제약 조건 (CAR이 개선 제안 시 반드시 고려)

- **인원:** 회장 1명 + AI 에이전트. (회의록·다인 협업 도구는 과잉.)
- **예산:** $0 · pre-revenue. (유료 도구는 매출 발생 시 — 지금 도입은 음의 수익.)
- **회장 무키워드 원칙:** 회장은 절차 키워드를 말할 필요 없다. Alpha가 전부 자동 적용.
- **회장 PC 경량 유지:** 불필요한 상주/자동시작 블로트 금지. 새 상주는 비용 대비 따져라.
- **헌법 비대화 금지:** CLAUDE.md를 계속 늘리지 말 것. 디테일은 전문가 파일로.
- **반(反)중간계층:** C레벨 라우터·깊은 피라미드 부활 금지(2026-06-12 폐기). 모든 위임은 1홉.

## 8. CAR이 개선할 수 있는 것 vs 절대 못 하는 것

| 영역 | CAR 권한 |
|------|----------|
| 도구(스킬·MCP) 추가 | 신뢰출처+1K+면 자동, 그 외 승인 큐 |
| 조직 구조·역할·라우팅·게이트 개선 | **제안만(propose-only).** 회장/Alpha 승인 후 적용 |
| CLAUDE.md(헌법)·settings.json 수정 | **직접 수정 금지.** 제안서 → 회장 승인 → Alpha가 반영 |
| 새 에이전트/전문가 신설 제안 | 제안만. 승인 후 Alpha가 페르소나 작성 |
| 에이전트 "교육"(프롬프트·체크리스트 개선) | 제안만. 검증 게이트·도메인 주입 그대로 적용 |

> **불변 경계:** CAR의 모든 조직 변경 = §1.5 도메인 주입 + 검증 하드게이트 + CAE 감사 대상. "CAR이 했으니 면제"는 없다. 헌법 자기개정 폭주를 구조로 차단 — CAR은 *제안*, 회장/Alpha가 *처분*.
