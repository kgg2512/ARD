# 비전 vs 현실 — ARD 자율 학습 루프

> 회장 원래 의도(2026, ARD 창설 계기): **"Claude가 영상을 보고, 유튜브 내용을 에이전트들이 이해하고, 알아서 정리·적용한다."**
> 이 문서는 그 비전과 현재 구현(`influencer_monitor.py`)의 갭을 코드 기준으로 정직하게 적는다.

---

## 1. 현재 실제로 하는 일 [확인 — 코드 직접 판독]

`influencer_monitor.py`(282줄)는 SessionStart 하루 1회 실행되며:

1. 4개 유튜브 채널의 **RSS 피드**를 가져온다 (`feeds/videos.xml`)
2. RSS의 **제목 + 설명 텍스트만** 본다 — 영상도, 자막도, 음성도, 화면도 **안 본다**
3. 제목/설명에 키워드(`claude code`, `mcp server` 등)가 있으면 "관련 영상"으로 표시
4. 정규식으로 `npx skills add <X>` 또는 `<X>-mcp` 같은 **패키지명 문자열**을 추출
5. 추출된 패키지를 자동 설치(스킬) / `settings.json`에 추가(MCP)

**즉 현재 = "RSS 텍스트 정규식 스크래퍼".** Claude(LLM)는 이 파이프라인에서 **단 한 번도 호출되지 않는다.** 훅이 돌리는 순수 파이썬일 뿐.

---

## 2. 비전과의 갭 (4단계 중 어디까지 왔나)

| 단계 | 비전 | 현재 | 상태 |
|------|------|------|------|
| ① 시청/지각 | 영상을 본다 | RSS 제목·설명 텍스트만. 자막조차 안 봄 | ❌ 없음 |
| ② 이해 | 에이전트가 내용을 이해 | 키워드 매칭 + 정규식. LLM 추론 0 | ❌ 없음 |
| ③ 정리 | 알아서 구조화·요약 | 없음. 패키지명만 추출 | ❌ 없음 |
| ④ 적용 | 배운 걸 적용 | "텍스트에 적힌 패키지명"만 설치 | 🟡 협소하게만 |

**결론:** 현재 구현은 회장 비전의 **약 10~15%만 달성** [추정]. 핵심인 ①시청 ②이해가 통째로 비어 있다. 영상 제목/설명에 패키지명이 문자로 안 적혀 있으면(대부분 그렇다) 아무것도 못 잡는다. "이 영상이 소개한 워크플로우/개념"은 원천적으로 이해 불가 — 정확히 회장이 원했던 그 부분.

---

## 3. 왜 이렇게 됐나 [추정]

훅은 빠르고 토큰 0이어야 해서, LLM 비호출 경량 스크립트로 만든 결과로 보인다. RSS 정규식은 싸지만 "이해"를 못 한다. **속도·비용 ↔ 이해력의 트레이드오프에서 비용 쪽으로 완전히 치우친 설계.**

⚠️ 추가로, 현재 **실제 가동 성과는 이 원격 환경에서 [확인] 불가** — 설치 이력 로그(`~/.claude/data/influencer_monitor_log.jsonl`)가 회장 로컬에만 있다. 지금까지 자동 설치가 몇 건 일어났는지는 회장 노트북에서 그 파일을 봐야 안다.

---

## 4. 비전 실현 로드맵 (현실적·$0 우선)

| Phase | 내용 | 도구 | 비용 |
|-------|------|------|------|
| **1 (현재)** | RSS 키워드 스크래퍼 — 명시적 패키지명만 | 표준 라이브러리 | $0 |
| **2 — 이해의 시작** | 신규 영상의 **자막(transcript) 수집** → 큐 파일 적재 → SessionStart 때 **Claude가 읽고 요약·판단** (여기서 비로소 LLM 개입) | `youtube-transcript-api` 등 | $0 (자막은 무료) |
| **3 — 정리·적용** | Claude가 자막 이해 → "이 영상이 소개한 워크플로우/스킬/MCP/패턴"을 구조화 추출 → ARD 노트로 정리 + 적용 후보 제시(자동설치는 신뢰 소스만, 나머진 회장 승인) | Claude + 기존 스킬 큐레이터 | $0 |
| **4 — 멀티모달(완전체)** | 영상 화면·데모까지 본다(비전 원안) | 영상 분석 MCP(예: higgsfield video_analysis — 임의 유튜브 적용 가능 여부 [추정·검증 필요]) | 유료 가능성 |

**가장 큰 도약은 Phase 2.** 자막 한 단계만 추가하면 "텍스트 매칭"이 "Claude의 이해"로 바뀐다. 자막은 무료이고, 대부분의 교육 채널(All About AI·Dave Ebbelaar 등)은 자막 품질이 좋다.

---

## 5. 정직한 한 줄

> **현재 ARD는 "유튜브를 보는 시스템"이 아니라 "유튜브 RSS 제목에서 패키지명을 줍는 스크립트"다.** 회장의 원래 비전(보고·이해하는 에이전트)은 아직 시작 안 됐다. Phase 2(자막→Claude 이해)부터가 진짜 ARD다.

---

## 6. 회장 결정 필요

Phase 2 구현(자막 파이프라인 + Claude 이해 단계)은 외부 라이브러리·네트워크 정책·새 훅이 얽힌 **실질 코딩 작업(T4급)**이다. 진행하려면 회장 "go"가 필요하다. 진행 시 검증 하드게이트·도메인 주입 그대로 적용.
